Утилита для обновления sequence'ов для для субд PostgreSql
Если в переменных среды (Пуск->Панель управления-> Система -> Свойста системы -> Переменные среды) в переменной PATHTEXT не стоит расширение .COM, то запускать утилиту следует от имени администратора.
Перед запуском рекомендуется добавить строку в файле pg_hba.conf (~PostgreSQLDir\data.pg_hba.conf)
# IPv4 local connections:
"host    all             %username%        127.0.0.1/32            trust"
это необходимо сделать для того, чтобы при запуске скрипта не запрашивался пароль.

Перед запуском следует убедиться, что все пути до приложения psql.exe, имя БД, хост, имя юзера указаны верно.
